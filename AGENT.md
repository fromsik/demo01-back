# AGENT.md - 구현/리뷰 시 방향성 가이드

## 목적
이 저장소의 변경은 항상 `PRD.md`를 기준으로 판단한다. 구현 상세가 모호하면 아래 우선순위를 따른다.

## 의사결정 우선순위
1. 보안/접근제어 (웹 로그인 필수, 사용자 데이터 격리)
2. 도메인 규칙 (목표 1~5, 10분 단위, 시간 겹침 금지)
3. API 일관성 (리소스 네이밍, HTTP status, 에러 메시지)
4. 단순성 (MVP 범위 우선, 과도한 추상화 지양)

## 백엔드 코드 규칙
- 입력 검증은 라우트 진입 시 즉시 수행.
- 에러 메시지는 사용자 액션 가능하도록 작성.
- 시간/정렬 규칙은 유틸 함수로 중앙화.
- 템플릿과 일별 계획 데이터는 분리 저장.

## 리뷰 체크리스트
- [ ] 인증 없는 요청 차단 여부
- [ ] 사용자 소유 데이터만 조회/수정 가능한지
- [ ] 목표 개수/시간 단위/겹침 검증 테스트 존재 여부
- [ ] MVP 범위를 벗어난 기능이 섞이지 않았는지
